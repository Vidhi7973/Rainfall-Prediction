Model Diagnostics & Data Verification

This section answers two critical questions: (1) Are the models genuinely learning from data, or predicting randomly? And (2) Is the dataset itself behaving as meteorological science expects? All numbers are derived from the real Rainfall.csv dataset.

Dataset Class Balance

Shows the ratio of rainy to non-rainy days in the training data. A heavily skewed dataset can bias models to predict only the majority class.

—
Accuracy vs Baseline (Majority-Class Guesser)

A "dumb" baseline that always predicts the most common class. If our ML models don't significantly beat this, they aren't adding value.

—
ROC-AUC Score per Model 0.5 = random · 1.0 = perfect

AUC (Area Under the ROC Curve) measures how well a model separates rain from no-rain days. Above 0.7 is good; above 0.9 is excellent.

ROC Curves — True Positive Rate vs False Positive Rate Diagonal = random classifier

Each curve shows the trade-off between catching real rain (TPR) and raising false alarms (FPR) as the decision threshold varies. Curves bowing toward the top-left corner indicate better performance. The dashed diagonal represents a random-guess classifier.

Confusion Matrices — What each model actually predicted

A confusion matrix breaks down predictions into four categories: True Negatives (correctly predicted No Rain), True Positives (correctly predicted Rain), False Positives (predicted Rain when it didn't rain — false alarm), and False Negatives (missed rain days — the most costly error in forecasting).

5-Fold Cross-Validation AUC — Checks for Overfitting High mean + low std = stable generalisation

The dataset is split into 5 equal folds. Each model trains on 4 folds and is tested on the 5th, rotating until every fold is tested. A high mean AUC with low standard deviation confirms the model generalises well to unseen data and isn't just memorising the training set.

Sensitivity vs Specificity Sensitivity = catches real rain · Specificity = avoids false alarms

Sensitivity (Recall) — the proportion of actual rainy days the model correctly identified. Critical for weather forecasting where missing rain is costly. Specificity — the proportion of dry days correctly identified as dry. High specificity means the model avoids unnecessary false rain alerts.

Precision-Recall Curves Higher area = better · Dashed = random baseline

Unlike ROC curves, PR curves focus specifically on the positive (Rain) class. Useful when the dataset is imbalanced. The area under this curve (AUPRC) rewards models that stay precise while also catching most rain days.

Calibration Curves — Predicted vs Actual Probability Closer to diagonal = better calibrated

Checks whether the model's stated probabilities are trustworthy. If a model says "70% chance of rain", it should actually rain about 70% of the time. Points on the diagonal = perfectly calibrated. Points above the diagonal = model is underconfident; below = overconfident.

Feature → Rain Tomorrow Correlations Point-biserial correlation · * = statistically significant (p < 0.05)

Measures how strongly each weather feature is linearly associated with rain tomorrow. Blue bars = positive correlation (feature increases → more rain likely). Orange bars = negative correlation (feature increases → less rain likely). Features marked with * are statistically significant — their correlation is unlikely to be due to chance.

Data Verification — Feature Distributions & Correlation Direction Checks that each feature correlates with rain in the expected meteorological direction

Each card shows a feature's correlation with rain tomorrow and whether it aligns with what meteorology predicts. Green = correlation direction is as expected (e.g. high humidity → more rain ✓). Amber = correlation is too small to be statistically significant. Red = direction is opposite to expectation — possible data anomaly worth investigating.
